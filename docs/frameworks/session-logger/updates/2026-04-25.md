# Session Logger Framework Update 2026-04-25

framework: session-logger
framework_update_date: 2026-04-25

## Summary

Adds repo-level project indexing, guided project selection on `start`, immediate `init project` activation, and primary/read-only multi-project activation semantics.

## What Changed

- Added `.agents/projects-index.json` as the shared project registry.
- Added short project refs derived from the first segment of each full UUID project ID.
- Added `index projects`, `list projects`, `init project <name>`, `deactivate`, and `deactivate all`.
- Added `activate <primary-ref> + <allowed-ref>, <allowed-ref>` syntax.
- Added `activate + <allowed-ref>, <allowed-ref>` syntax for adding read-only projects without changing the current primary.
- Added confirmation behavior before replacing an existing primary with `activate <project-ref>`.
- Defined primary projects as Session Logger read/write scopes.
- Defined allowed projects as explicitly read-only scopes.
- Allowed `start` with no active project to use registry metadata to help the user choose a project.

## Why It Changed

The previous start flow assumed the user already knew which project to activate. The new registry gives Session Logger a safe, non-memory surface for suggesting likely projects without weakening default-deny project isolation.

## Impact On Project Structure

Projects may store their immutable full UUID registry ID in project-local Session Logger config, but `.agents/projects-index.json` is the canonical registry and also stores short convenience refs.

## Impact On Governed Files

Projects that adopt this update should align their workflow start procedure with registry-assisted project selection and primary/read-only activation language.

## Impact On Governed Data

Session memory remains project-local. The registry stores project metadata and recent activation history only; it is not live workflow memory.

## Required Audit Checks

- Confirm `.agents/projects-index.json` exists and is valid JSON when the workspace uses project indexing.
- Confirm full project IDs are unique.
- Confirm short project refs are unique and match the first UUID segment.
- Confirm workflow start docs do not read project memory before activation.
- Confirm allowed read-only projects cannot receive Session Logger writes, checkpoints, framework upgrades, commits, or session-end logs.

## Possible Upgrade Actions

- create `.agents/projects-index.json` with registered initialized projects
- update `docs/workflow/session_start.md` to describe registry-assisted project selection
- update project-local Session Logger docs to distinguish primary read/write and allowed read-only projects
