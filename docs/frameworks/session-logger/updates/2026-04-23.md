# Session Logger Framework Update 2026-04-23

framework_update_date: 2026-04-23

## Summary

Added the first manual framework audit procedure for Session Logger and formalized project-local framework audit state.

## Why This Change Exists

Session Logger needed a safe way to compare a project's adopted framework state with the current shared framework methodology without introducing background auto-checks or silent upgrade behavior.

## Impact On Project Structure

- projects using Session Logger now have a framework-facing config file at `.codex/session-logger.toml`
- manual audit is treated as part of the framework lifecycle between activation and deeper work

## Impact On Governed Files

- `.codex/session-logger.toml` may need to exist
- `docs/workflow/` procedure and scaffold files may need to be reviewed against the current framework wording
- project-local Session Logger scaffold copies may need updates when framework procedure text changes

## Impact On Governed Data

- project adopted framework date is now tracked explicitly
- last manual audit date and status are tracked explicitly
- no workflow memory semantics change in this update

## Required Audit Checks

- does the project have `.codex/session-logger.toml`?
- does the config store an adopted framework date?
- do governed workflow files exist where the framework expects them?
- do project-local Session Logger procedure files materially match the current framework procedure?

## Possible Upgrade Actions

- create `.codex/session-logger.toml` with the adopted framework date
- update governed workflow files to current framework procedure wording
- add any missing governed workflow scaffold files
- update manual audit status in the project config after review
